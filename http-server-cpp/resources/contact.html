Test POST Requests

You can test the JSON upload functionality by sending POST requests to the server. Here's an example:

Example JSON for Upload:

{ "name": "Test User", "email": "test@example.com", "message": "This is a test message for the HTTP server", "timestamp": "2024-03-15T10:30:00Z", "data": { "type": "test", "value": 42 } }

How to Test:

  • Use curl: curl -X POST -H "Content-Type: application/json" -d '{"test": "data"}' http://localhost:8080/upload
  • Use Postman or similar API testing tools
  • Send POST request with JSON body to any endpoint

Server Testing

This server supports various test scenarios:

  • GET Requests: HTML files, binary downloads (images, text)
  • POST Requests: JSON data uploads to /uploads/ directory
  • Security Tests: Path traversal protection, host validation
  • Concurrency Tests: Multiple simultaneous connections

Server Statistics

The server provides comprehensive logging including:

  • Request/response details with timestamps
  • Thread pool status and connection counts
  • File transfer statistics and binary integrity
  • Security violation logging
